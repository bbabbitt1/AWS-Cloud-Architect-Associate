AWS Machine Learning Services

Overview

AWS Machine Learning services span from pre-trained AI models (no data science expertise required) to fully managed platforms for building custom models. Solutions Architects use ML services to add intelligence to applications: image recognition, natural language processing, personalization, search, forecasting, and more. This section covers key AWS AI/ML services and exam scenarios.

Amazon Rekognition — Image & Video Analysis

Service Overview

Amazon Rekognition is a pre-trained machine learning service that analyzes images and videos without requiring data science expertise. It uses deep learning models trained on billions of images to identify objects, people, text, scenes, activities, and more. Rekognition integrates seamlessly with S3, Lambda, and other AWS services for end-to-end vision pipelines.

Core Capabilities

  • Object & Scene Detection: Identify objects (car, dog, tree), activities (person running, playing), and scenes (beach, cityscape, forest) in images with confidence scores.
  • Text Detection & Recognition (OCR): Extract text from images. Useful for document processing, signage reading, receipt digitization.
  • Facial Analysis: Detect faces and extract attributes (age range, gender, emotion, glasses, beard, etc.). No face identification required—purely structural analysis.
  • Facial Recognition & Verification: Compare a face against a database of known faces (face collection). Determine if two faces belong to the same person. Useful for user verification without passwords.
  • Celebrity Recognition: Identify celebrities in images (works for celebrities in entertainment, politics, sports).
  • Video Analysis: Same capabilities applied to video streams. Detect when people enter/exit frames, when objects appear, segment videos by scene changes.

Content Moderation

Rekognition can flag inappropriate, unwanted, or offensive content automatically. Set confidence thresholds for each category (explicit nudity, violence, drugs, tobacco, etc.).

  • Use Cases: Social media platforms moderate user uploads in real-time. E-commerce sites ensure product photos comply with content policies. Broadcast networks filter ads and content. Dating apps verify user photos.
  • Workflow: User uploads image → Lambda triggered → Rekognition analyzes → if confidence(inappropriate) > threshold, flag for manual review (using Amazon Augmented AI) or reject automatically.
  • Amazon Augmented AI (A2I) Integration: For borderline cases (confidence 0.5-0.8), route to human reviewers for final decision. A2I manages the review workflow and consensus logic.

Face Collection & Facial Search Database

Build searchable databases of faces for verification and identification.

  • Face Collection: Store face vectors (mathematical representations) in a collection. Index millions of faces efficiently.
  • Facial Search: Upload a photo → Rekognition finds matching faces from the collection. Returns matches with similarity confidence.
  • Exam Scenario: "Build a mobile app where users verify identity by taking a selfie" → Rekognition facial search against enrolled faces. Or "Detect if a person in security footage matches watchlist" → Compare video frames against face collection.

Use Cases (from Exam Perspective)

  • Photo Organization: Automatically tag photos by objects/scenes without manual tagging.
  • Content Moderation: Flag inappropriate content in social media, dating apps, or user-generated content platforms.
  • Security & Verification: Facial recognition for user authentication in mobile apps or physical security systems.
  • Media Analysis: Analyze sports footage to track player positions, ball location, or game events. Analyze news footage for scene changes and speaker identification.
  • Retail Analytics: Analyze in-store video to count customers, measure dwell times, detect if specific products are picked up.
  • Exam Pattern: "Automatically analyze images for content," "detect objects in video," "identify people" → Rekognition.

Amazon Transcribe — Automatic Speech Recognition

Service Overview

Amazon Transcribe converts speech to text using deep learning automatic speech recognition (ASR) models. It supports multiple languages, identifies speakers, and automatically removes personally identifiable information (PII) from transcripts. Useful for transcribing meetings, customer service calls, media content, and voice applications.

Key Features

  • Automatic Speech Recognition: Converts audio files (MP3, WAV, Ogg, Flac, etc.) or streaming audio to text. Works offline (batch) or in real-time (streaming).
  • Multi-Language Support: Automatic language identification. Transcribe in 100+ languages simultaneously in multilingual audio.
  • Speaker Identification: Automatically identify and label different speakers in audio. Useful for meeting transcripts ("Speaker 1: ...", "Speaker 2: ...").
  • PII Redaction: Automatically detect and remove personally identifiable information (phone numbers, email addresses, credit card numbers, etc.) from transcripts. Critical for compliance (HIPAA, PCI-DSS).
  • Custom Vocabulary: Add industry-specific terms or proper nouns that default models might misrecognize. E.g., product names, company names, technical jargon.
  • Confidence Scores: Each word transcribed includes a confidence score. Use scores to identify uncertain portions for human review.

Architecture Pattern: Transcribe → Comprehend Pipeline

Combine Transcribe with Comprehend for powerful text analysis workflows:

  • Step 1: Customer calls contact center → audio recorded → Transcribe converts to text.
  • Step 2: Comprehend analyzes transcript for sentiment (positive/negative/neutral), entities (product names, customer names), key phrases.
  • Step 3: Results stored in DynamoDB; dashboards show call sentiment trends, common issues, customer satisfaction metrics.
  • Exam Pattern: "Analyze customer calls to understand sentiment and issues" → Transcribe + Comprehend pipeline.

Use Cases

  • Contact center quality assurance (analyze calls for compliance, agent performance).
  • Closed captions for videos (accessibility).
  • Meeting transcription and searchable meeting minutes.
  • Medical documentation: Transcribe dictated medical notes; Comprehend Medical extracts diagnoses and medications.
  • Podcasts and webinars: Generate transcripts for search and accessibility.
  • Exam Pattern: "Convert audio to text," "transcribe calls," "speech recognition" → Transcribe.

Amazon Polly — Text-to-Speech Synthesis

Service Overview

Amazon Polly converts written text to realistic, natural-sounding speech. It uses advanced text-to-speech (TTS) technology to synthesize speech in multiple languages with various voices (different accents, ages, genders). Polly can create audio files or stream speech in real-time for interactive applications.

Key Features

  • Multiple Languages & Voices: 20+ languages with 100+ natural voices (male, female, different accents). Select voice, language, and speaking style (conversational, news-like, etc.).
  • SSML Support (Speech Synthesis Markup Language): Control speech output with SSML tags for prosody, emphasis, pauses. Example: <speak>The price is <emphasis>five dollars</emphasis></speak> emphasizes "five dollars." SSML enables natural-sounding speech customization.
  • Pronunciation Lexicons: Define custom pronunciations for words/acronyms. E.g., ensure "SQL" is pronounced "sequel" not "s-q-l," or specify how your company name is pronounced.
  • Real-Time Streaming: Stream synthesized speech in real-time instead of generating full audio files. Useful for interactive applications (chatbots, voice assistants).
  • Neural Voices: Advanced neural TTS provides more natural, lifelike speech compared to standard voices.

Use Cases

  • Accessibility: Read web content aloud for visually impaired users.
  • Voice-Enabled Applications: Chatbots, virtual assistants, voice-based interfaces.
  • Content Creation: Generate audio versions of blog posts, articles, documentation.
  • Contact Centers: IVR (Interactive Voice Response) systems with natural-sounding prompts.
  • Learning Platforms: Create audio versions of educational content.
  • Exam Pattern: "Convert text to speech," "voice synthesis," "audio content generation" → Polly.

Amazon Translate — Machine Translation

Service Overview

Amazon Translate automatically translates text from one language to another using neural machine translation (NMT). It supports 100+ languages and language pairs. Translate integrates with other AWS services to build multilingual applications at scale.

Key Features

  • Neural Machine Translation: Uses deep learning for accurate, contextual translations. Better than older statistical translation models.
  • Batch Translation: Translate large volumes of documents stored in S3. Translate processes files asynchronously and stores translated output in S3.
  • Real-Time API: Translate text on-demand via API for interactive applications.
  • Custom Terminology: Define domain-specific terms and their translations. E.g., ensure company-specific jargon translates correctly. Improves consistency across documents.
  • Language Detection: Automatically detect source language if not specified.

Use Cases

  • Multilingual Websites: Translate content to serve users in different regions.
  • Global Customer Support: Translate customer inquiries and support responses.
  • Content Localization: Translate documents, user manuals, help documentation.
  • International E-Commerce: Translate product descriptions, reviews.
  • Exam Pattern: "Translate content to multiple languages," "multilingual application" → Translate.

Amazon Lex & Amazon Connect

Amazon Lex — Conversational AI

Amazon Lex builds conversational interfaces (chatbots, voice assistants) using automatic speech recognition (ASR) and natural language understanding (NLU). Lex powers Amazon Alexa and can be integrated into applications.

  • Intents: Represent what users want (book hotel, check balance, report fraud).
  • Slots: Extract parameters from user input (date, amount, account number). Lex collects slot values through conversation.
  • Fulfillment: After collecting required slots, trigger a Lambda function or external service to fulfill the request (book hotel, transfer money, etc.).
  • Integration: Lex bots integrate with messaging platforms (Slack, Facebook Messenger), web applications, phone systems (Amazon Connect).
  • Use Cases: Bank chatbots for account inquiries, e-commerce shopping assistants, IT helpdesk bots, appointment booking.
  • Exam Pattern: "Build chatbot," "conversational interface," "voice application" → Lex.

Amazon Connect — Cloud Contact Center

Amazon Connect is a cloud-based contact center platform enabling customer service, sales, and support via phone, chat, or video. It integrates with Lex for chatbots, Polly for voice prompts, Transcribe for call recording, and Comprehend for sentiment analysis.

  • Omnichannel Support: Handle customer inquiries via phone, chat, email, video in a unified interface.
  • IVR Integration: Use Lex-powered chatbots as IVR (Interactive Voice Response) to route calls, answer common questions, or collect information before routing to agents.
  • Agent Workspace: Unified dashboard showing customer history, previous interactions, recommended responses.
  • Call Recording & Analysis: Transcribe calls with Transcribe, analyze sentiment with Comprehend for quality assurance.
  • Analytics: Track contact metrics (call duration, wait time, customer satisfaction), generate reports.
  • Exam Pattern: "Build cloud-based contact center," "customer service platform" → Connect. Often combined with Lex for intelligent IVR.

Amazon Comprehend — Natural Language Processing

Service Overview

Amazon Comprehend uses natural language processing (NLP) to extract insights from text. It identifies key phrases, entities (people, places, organizations), sentiment, and language. Comprehend processes batch documents or streaming text in real-time.

Core Capabilities

  • Sentiment Analysis: Determine if text expresses positive, negative, neutral, or mixed sentiment. Useful for analyzing customer reviews, social media, feedback.
  • Entity Recognition: Identify named entities (people, organizations, locations, dates, quantities, commercial items). E.g., from "John Smith works at Amazon in Seattle," extract: Person=John Smith, Organization=Amazon, Location=Seattle.
  • Key Phrase Extraction: Identify important phrases/concepts in text. E.g., from "The laptop has excellent battery life," extract "battery life" as a key phrase (product feature).
  • Language Detection: Identify the language of input text (if not known).
  • Syntax Analysis: Part-of-speech tagging (noun, verb, adjective, etc.).
  • Targeted Sentiment: Analyze sentiment about specific aspects (e.g., "Screen is great, but battery sucks" → positive sentiment about screen, negative about battery).

Batch Processing & Real-Time Streaming

  • Batch Analysis: Upload documents to S3 → Comprehend processes asynchronously → results written to S3. For large-scale analysis (thousands of documents).
  • Real-Time API: Call Comprehend API for immediate analysis. For low-latency requirements or small document volumes.
  • Streaming Integration: Integrate with Kinesis Data Firehose to analyze continuous text streams (e.g., customer feedback, social media monitoring).

Use Cases

  • Customer Review Analysis: Automatically classify reviews as positive/negative, identify complaints, track sentiment trends.
  • Social Media Monitoring: Analyze tweets/posts for brand sentiment, identify influencers, detect crises.
  • Document Classification: Automatically categorize support tickets, emails, or documents for routing/prioritization.
  • Resume/Job Matching: Extract skills and experience from resumes; match against job descriptions.
  • Compliance & Audit: Scan documents for sensitive content, entities, compliance-related phrases.
  • Exam Pattern: "Analyze text for sentiment," "extract information from documents," "natural language understanding" → Comprehend.

Amazon Comprehend Medical

Service Overview

Amazon Comprehend Medical is specialized for healthcare text analysis. It extracts medical information (diagnoses, medications, symptoms, dosages, procedures) from clinical text. Critical for healthcare organizations to automate clinical documentation, billing, and research.

Key Capabilities

  • Medical Entity Extraction: Identify medical entities (diagnoses, medications, procedures, symptoms, allergies) with confidence scores.
  • Relationship Detection: Understand relationships (medication dosage, procedure indication). E.g., from "Prescribe 500mg Aspirin for heart pain," extract: medication=Aspirin, dosage=500mg, indication=heart pain.
  • Protected Health Information (PHI) Detection: Identify sensitive health information (patient names, medical record numbers, contact details).
  • Medical Linking: Normalize medical terms to standardized medical ontologies (SNOMED CT, RxNorm). Enables data standardization and interoperability.

Data Sources & Integration Patterns

  • Batch Analysis from S3: Store clinical documents (PDF scans, text exports) in S3 → Comprehend Medical processes batch → results stored in S3.
  • Real-Time with Transcribe: Patient visits recorded → Transcribe converts audio to text (removes PII automatically) → Comprehend Medical extracts diagnoses, medications → results sent to EHR system for physician review.
  • Streaming from Kinesis Firehose: Clinical notes streamed in real-time → Firehose buffers → triggers Comprehend Medical for processing → results stored in S3 or DynamoDB for downstream analysis/visualization.

Use Cases

  • Automated Clinical Documentation: Extract structured data from physician notes for EHR systems.
  • Clinical Trial Matching: Find patients matching trial criteria by analyzing their medical records.
  • Healthcare Analytics: Analyze disease prevalence, medication effectiveness, treatment outcomes.
  • Medical Billing & Coding: Automatically extract diagnosis/procedure codes for billing.
  • Exam Pattern: "Analyze medical documents," "extract clinical information," "healthcare compliance" → Comprehend Medical.

Amazon SageMaker — Fully Managed Machine Learning Platform

Service Overview

Amazon SageMaker is a fully managed machine learning platform for building, training, and deploying machine learning models. It abstracts away infrastructure complexity, allowing data scientists and developers to focus on model development. SageMaker integrates with AWS services (S3, Lambda, CloudWatch, etc.) for end-to-end ML pipelines.

Core Components & Workflow

  • Data Preparation: SageMaker Studio provides Jupyter notebooks for exploratory data analysis. AWS Glue can preprocess and clean data before model training.
  • Feature Engineering: Transform raw data into features suitable for ML models. SageMaker Processing jobs can run data processing scripts (Python, Spark) at scale.
  • Model Training: SageMaker Training jobs automatically provision compute resources (EC2 instances, GPUs), run training scripts, manage distributed training across multiple instances. Supports built-in algorithms (Linear Learner, XGBoost, etc.) or custom Docker containers.
  • Hyperparameter Tuning: Automatically search for optimal model hyperparameters by running multiple training jobs with different parameter combinations.
  • Model Evaluation & Validation: Assess model performance on validation/test datasets. Use SageMaker Experiments to track and compare different model versions.
  • Model Deployment: Deploy trained models to SageMaker Endpoints (managed, auto-scaling inference servers). Endpoint automatically provisions servers, manages traffic, handles failover.
  • Monitoring & Management: CloudWatch integration monitors endpoint metrics (latency, invocations, errors). Model Monitor detects data drift and model performance degradation over time.

SageMaker Features & Capabilities

  • Autopilot: Automatically build, train, and tune ML models from data with minimal code. Ideal for non-ML experts.
  • Canvas: No-code interface for building ML models visually. Drag-and-drop data, automatically trains models.
  • Feature Store: Centralized repository for ML features. Share engineered features across teams, ensure consistency, handle feature versioning.
  • Pipelines: Orchestrate ML workflows (data preparation → training → evaluation → deployment) as repeatable, automated pipelines.
  • Built-in Algorithms: Pre-implemented algorithms optimized for AWS (XGBoost, Linear Learner, Factorization Machines, etc.). No need to implement from scratch.
  • Marketplace Models: Pre-trained models available from AWS and third-party providers. Fine-tune for your data instead of training from scratch.

Architecture Pattern: SageMaker End-to-End ML Pipeline

Scenario: E-commerce company wants to predict customer churn to proactively retain customers.

  • Step 1 (Data): Historical customer data (purchases, engagement, demographics) stored in S3.
  • Step 2 (Preparation): Glue job extracts data from S3, cleans missing values, encodes categorical variables → outputs to S3 as training dataset.
  • Step 3 (Training): SageMaker Training job loads training data from S3, trains XGBoost model using GPU instances, saves trained model to S3.
  • Step 4 (Evaluation): SageMaker Batch Transform job applies model to validation data, generates predictions, evaluates accuracy/precision/recall.
  • Step 5 (Deployment): Deploy model to SageMaker Endpoint. Endpoint auto-scales based on traffic.
  • Step 6 (Integration): Application calls Endpoint API for predictions. For each customer, get churn probability; if high, trigger retention workflow (offer discount, personalized recommendation).
  • Step 7 (Monitoring): Model Monitor detects data drift (current customer data distribution changes). If drift detected, trigger retraining job automatically.

Exam Scenarios & Patterns

  • "Build custom ML model from scratch" → SageMaker. Provides infrastructure and tools for full ML lifecycle.
  • "Use pre-trained model for image analysis" → Rekognition (no custom training needed).
  • "Forecast demand/revenue" → SageMaker with time-series algorithm (Forecast is specialized variant, but SageMaker can also do this).
  • "Build recommendation system" → SageMaker with Factorization Machines or Personalize service (if simpler personalization needed).

Amazon Kendra — Intelligent Search Service

Service Overview

Amazon Kendra is an intelligent search service powered by machine learning. Unlike keyword-based search, Kendra understands natural language queries and returns relevant answers from documents. It uses deep learning to comprehend context and retrieve the most relevant information, even if documents don't contain exact keywords.

Key Features

  • Natural Language Search: Users ask questions in natural language (e.g., "How do I reset my password?"). Kendra understands intent and returns relevant answers, not just keyword-matched documents.
  • Incremental Learning: Learn from user interactions (which results users clicked, which they ignored). Improve relevance over time automatically.
  • Multiple Data Sources: Index documents from S3, relational databases, internal wikis, FAQs, and more. Unified search across disparate sources.
  • FAQs & Document Snippets: Extract answers from FAQ documents or highlight specific passages within longer documents. Return concise answers, not full documents.
  • Security & Access Control: Respect document-level security (only return documents user has access to).
  • Faceted Search: Filter results by document attributes (date, author, department, etc.).

Use Cases

  • Enterprise Search: Employees search across company documentation, policies, procedures. Find answers without browsing entire manuals.
  • Customer Self-Service: Customers search knowledge base, FAQ, product documentation. Reduces support tickets.
  • Legal Discovery: Lawyers search through contracts, regulations, case files to find relevant legal precedents.
  • Healthcare Research: Researchers search medical literature, clinical guidelines, research papers.
  • Exam Pattern: "Search for answers in documents," "natural language search," "intelligent search" → Kendra.

Amazon Personalize — Real-Time Recommendations

Service Overview

Amazon Personalize builds real-time personalized recommendation engines without ML expertise. It uses collaborative filtering and deep learning to recommend products, content, or services based on user behavior and preferences. Personalize integrates into applications via APIs and can power recommendation widgets, email campaigns, and personalized experiences.

Key Concepts

  • User-Item Interactions: Feed Personalize with user behavior data (view, click, purchase, rating). Personalize learns patterns from interactions.
  • Recommendations: Given a user, Personalize recommends items they're likely to be interested in. Output: ranked list of recommended items with relevance scores.
  • Cold Start Problem: For new users with no interaction history, Personalize uses content-based filtering (recommend items similar to items they viewed) or popularity-based recommendations.
  • Real-Time Inference: Call Personalize API for on-demand recommendations. Sub-100ms latency for user-facing applications.
  • Batch Recommendations: Generate recommendations for all users asynchronously. Store results in S3 for use in campaigns/emails.

Use Cases

  • E-Commerce: "Customers who viewed this also viewed..." or "Recommended for you" sections. Increase product discovery and conversion.
  • Media & Entertainment: Recommend movies, shows, articles based on viewing history. Increase engagement and retention.
  • Retail & Marketing: Personalized email campaigns. Recommend products to customer segments.
  • Content Platforms: Recommend news articles, blog posts, podcasts.
  • Exam Pattern: "Build recommendation engine," "personalized product suggestions," "real-time recommendations" → Personalize.

Amazon Textract — Document Analysis & Data Extraction

Service Overview

Amazon Textract automatically extracts text, structured data, and relationships from scanned documents, PDFs, and images using machine learning. It identifies text, handwriting, tables, forms, and key-value pairs without manual processing or OCR setup.

Core Capabilities

  • Text Extraction (OCR): Extract text from scanned documents, PDFs, images. Recognizes both printed and handwritten text. Provides confidence scores for extracted text.
  • Table Extraction: Identify and extract tables with rows/columns preserved. Useful for financial statements, data tables, spreadsheets scanned as PDFs.
  • Form Processing: Extract key-value pairs from forms (e.g., "First Name: John", "Date: 12/25/2025"). Useful for processing application forms, medical records, loan applications.
  • Document Classification: Classify documents by type (invoice, receipt, resume, bank statement) automatically.
  • Signature Detection: Identify signatures on documents (useful for detecting signed vs. unsigned documents).
  • Relationship Detection: Understand relationships between extracted elements (e.g., which name belongs to which address on a form).

Integration Pattern: Textract + Comprehend + DynamoDB

Scenario: Insurance company receives thousands of claim forms daily (mixed printed and handwritten). Manual processing is slow and error-prone.

  • Step 1: Forms scanned and uploaded to S3 → Lambda triggered by S3 event.
  • Step 2: Lambda invokes Textract → extracts text, tables, key-value pairs → returns JSON with extracted data.
  • Step 3: Extract text (description, remarks) → Comprehend analyzes for sentiment or key phrases (fraud indicators).
  • Step 4: Store extracted structured data + Comprehend insights in DynamoDB → claims processed, indexed, searchable.
  • Step 5: Dashboard shows processed claims, flagged high-risk claims (low confidence or fraud indicators) for manual review.
  • Benefit: Reduces manual data entry time by 90%, improves accuracy, enables faster claims processing.

Use Cases

  • Document Processing: Digitize paper documents (forms, contracts, receipts, invoices) automatically.
  • Financial Services: Process mortgage applications, tax returns, financial statements.
  • Healthcare: Extract patient information from medical forms, process insurance claims.
  • Legal: Extract key information from contracts (parties, dates, terms).
  • Retail & Logistics: Process shipping labels, invoices, receipts at scale.
  • Exam Pattern: "Extract data from scanned documents," "digitize paper forms," "OCR at scale" → Textract.

Exam-Style Questions & Answers

  1. Q: A retail company wants to moderate user-generated product reviews for inappropriate content. Thousands of reviews are uploaded daily. Reviews must be analyzed within hours. Which AWS service automatically flags offensive content?

    A: Rekognition Content Moderation (for images) or Comprehend (for text). Since the scenario mentions reviews (text), Comprehend analyzes text for sentiment and could flag problematic content. However, if reviews include images, Rekognition detects inappropriate imagery. For text reviews, Comprehend is the primary tool. Set confidence thresholds; low-confidence flagged content goes to humans via Amazon Augmented AI.
  2. Q: A contact center receives customer calls in multiple languages. Calls must be transcribed, analyzed for sentiment, and routed intelligently. How would you architect this?

    A: (1) Amazon Connect records calls. (2) Transcribe converts audio to text with automatic language identification. (3) Comprehend analyzes sentiment (positive/negative/neutral). (4) Connect routes high-satisfaction calls to junior agents; low-satisfaction/frustrated calls to senior agents. (5) Store sentiment/language metadata in DynamoDB for analytics. (6) CloudWatch dashboard shows sentiment trends by language.
  3. Q: Your company scans thousands of invoices daily (mixture of printed and handwritten). Manual data entry is slow. Build an automated invoice processing pipeline.

    A: (1) Invoices uploaded to S3. (2) Lambda triggered → Textract extracts text, tables, key-value pairs (vendor, amount, date). (3) Comprehend analyzes description/notes for key phrases (payment terms, special instructions). (4) Lambda stores structured data in DynamoDB. (5) Finance system queries DynamoDB for processed invoices, flagged items go to humans for review. Benefit: 90% reduction in manual data entry, faster processing, fewer errors.
  4. Q: An e-commerce company wants to recommend products to each user. Recommendations should be real-time and personalized based on user behavior. Which AWS service?

    A: Amazon Personalize. Feed user-item interactions (clicks, purchases, ratings) → Personalize learns patterns → recommend items user is likely to buy. Integrate into application for real-time recommendations. Handles cold-start for new users. Compare with Kendra (search/answers), SageMaker (custom models).
  5. Q: Your company needs to build a custom ML model to predict equipment failure from sensor data. You have ML expertise but want to focus on the model, not infrastructure. Which AWS service?

    A: Amazon SageMaker. Fully managed ML platform abstracts infrastructure. Provision compute via SageMaker Training, deploy to Endpoints, monitor with CloudWatch. Supports custom code or built-in algorithms. Alternative: Forecast service if this is purely time-series prediction, but SageMaker is more flexible for custom models.
  6. Q: A bank wants to automate customer support with a chatbot that understands natural language and can answer account questions. Which AWS services?

    A: Amazon Lex + Lambda + DynamoDB. (1) Lex understands intents (check balance, transfer money, report fraud). (2) Lex extracts slots (account number, amount). (3) Lambda fulfillment function queries DynamoDB for account details, executes transaction. (4) Lex responds conversationally. Optional: integrate with Amazon Connect for phone/chat channels.
  7. Q: Your healthcare organization stores thousands of patient medical records. You need a system where doctors can search for medical information (diagnoses, medications, treatments) across all records. Keyword search is insufficient; results must be semantically relevant. Which service?

    A: Amazon Kendra. Indexes medical documents. Doctor asks "What medications are effective for diabetic patients?" → Kendra understands the intent and returns relevant documents/answers (not just documents containing "diabetes" and "medication"). Incremental learning improves relevance over time based on which results doctors actually use.
  8. Q: Your company converts blog posts to audio for accessibility and podcast distribution. Text-to-speech output must be natural-sounding with correct pronunciation of company terms (product names, acronyms). Which service?

    A: Amazon Polly. Supports SSML for prosody control and custom pronunciation lexicons. Define pronunciations for your specific terminology (company name, product names). Stream speech in real-time or generate audio files. Multiple voices and languages available. Neural voices provide the most natural output.
  9. Q: You're launching a global app and need to support 50+ languages. Users interact in any language. How do you handle multilingual support?

    A: Combine services: (1) Translate for content localization (translate UI, help docs). (2) Comprehend for language detection (automatically identify user's language). (3) Polly for voice output in user's language. (4) Transcribe with language detection for voice input. Glue them together in a Lambda function: detect language → translate content → synthesize in user's language.
  10. Q: A security team must analyze stored medical images to detect anomalies (potential tumors, fractures). Training a custom model would take months. What's the fastest approach?

    A: Use pre-trained models from SageMaker Marketplace or AWS services. For medical imaging, SageMaker Marketplace offers pre-trained radiology models. Fine-tune on your dataset (faster than training from scratch). Alternatively, if Rekognition supported medical imaging (it doesn't specifically), you'd use that. For custom medical models, SageMaker with healthcare dataset acceleration services.

Best Practices & Architecture Patterns

  • Pre-Trained vs Custom Models: Use pre-trained services (Rekognition, Comprehend, Textract) for common tasks. Build custom models (SageMaker) only when pre-trained models don't meet your requirements.
  • Pipeline Architecture: Combine services in pipelines (Textract → Comprehend → DynamoDB) for complex workflows. Lambda orchestrates transitions between services.
  • Batch vs Real-Time: Batch processing (Glue, SageMaker Batch Transform) for high-volume, asynchronous analysis. Real-time APIs (SageMaker Endpoints, Personalize, Kendra) for interactive applications.
  • Data Privacy: Use Transcribe's PII redaction, Comprehend Medical for PHI detection, KMS encryption for sensitive data. Ensure compliance (HIPAA, GDPR, etc.).
  • Cost Optimization: For variable workloads, use API-based services (Rekognition, Comprehend) that charge per request. For sustained workloads, SageMaker Endpoints or on-premises fine-tuning may be more cost-effective.
  • Monitoring & Feedback: Log predictions, collect feedback, retrain models periodically. Use Model Monitor for drift detection. Implement human-in-the-loop for edge cases (Amazon Augmented AI).

Closing Notes

AWS Machine Learning services span from simple pre-trained APIs (Rekognition, Comprehend) to full ML platforms (SageMaker). Exam scenarios test your ability to select the right service based on use case: image analysis → Rekognition, text analysis → Comprehend, speech → Transcribe/Polly, custom models → SageMaker, search → Kendra, recommendations → Personalize. Master service selection, understand integration patterns (Textract → Comprehend → DynamoDB), and practice combining services for end-to-end solutions. This skill is critical for Solutions Architect exams and real-world ML applications.
