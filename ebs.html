EC2 - Instance Storage

EC2 Instance Storage

EC2 instances can use different types of storage options, each with its own characteristics and use cases. The primary storage options for EC2 instances include:

  • Instance Store (Ephemeral Storage):
    • When high-performance hardware disk is needed! Great I/O performance.
    • Temporary storage that is physically attached to the host computer.
    • Caveat - Data is lost when the instance is stopped or terminated.
    • Best suited for temporary data, caches, or scratch space.
  • Elastic Block Store (EBS):
    • Persistent block storage that can be attached to EC2 instances.
    • Bound to a specific AZ
    • Data persists independently of the instance's lifecycle. Can be detached and attached to EC2 instances very easily.
    • Supports features like snapshots, encryption, and resizing.
    • EBS Snapshots:
      • Move to an archive tier for 75% savings.
      • Recycle bin Rules to retain deleted snapshots
      • Fast Snapshot Restore - force full initialization of snapshot to have no latency on the first use.
      • Point-in-time backups of EBS volumes.
      • Stored in S3 for durability.
      • Can be used to create new EBS volumes or restore existing ones.
    • Multi-Attach
      • Attach a single io1 or io2 volume to multiple EC2 instances within the same AZ.
      • Useful for clustered applications that require shared storage.
      • Achieve higher application availability in clustered Linux applications OR applications that must manage concurrent.
      • Does not allow for attaching across AZs
      • Up to 16 EC2 Instances at a time.
    • EBS Encryption
      • Data at rest, data in transit, and all snapshots are encrypted.
      • Uses AWS KMS for key management.
      • No performance impact when using encryption.
      • Copying an unencrypted snapshot allows encryption.
      • Snapshots of encrypted volumes are encrypted.
      • Encrypting Unencrypted EBS Volumes
        • Create a snapshot of the unencrypted volume.
        • Copy the snapshot and enable encryption during the copy process.
        • Create a new encrypted volume from the encrypted snapshot.
        • Replace the unencrypted volume with the new encrypted volume on the EC2 instance.
    • EBS Volume Types:
      • Note: Only gp2/gp3 and io1/io2 Block Express can be used as boot volumes.
      • General Purpose SSD (gp2, gp3)
        • Balanced price and performance for a wide variety of workloads.
        • Baseline performance with the ability to burst.
        • gp3
          • Higher baseline performance than gp2.
          • Allows independent scaling of IOPS and throughput.
        • gp2
          • Performance scales with volume size.Max IOPS is 16,000
      • Provisioned IOPS SSD (io1, io2)
        • High-performance SSD designed for I/O-intensive applications.
        • Provisioned IOPS for consistent performance.
        • Great for databases workloads (sensitive to storage perf and consistency)
      • Throughput Optimized HDD (st1)
        • Low-cost HDD designed for frequently accessed, throughput-intensive workloads.
      • Cold HDD (sc1)
        • Lowest cost HDD designed for less frequently accessed data.
      • Magnetic (standard) - legacy
        • Previous generation magnetic storage.
  • Elastic File System:
    • MANAGED(!) Object storage service that can be accessed from EC2 instances.
    • Ideal for storing large amounts of unstructured data.
    • Provides high durability and availability.
    • 3x cost of EBS
    • Can be mounted concurrently on multiple EC2 instances.
    • Use Cases:
      • Content management, web serving, data sharing, and home directories.
    • Only compatible with Linux AMI - not windows
    • Scales automatically!
    • EFS Scale
      • Storage scales from GBs to PBs automatically as files are added or removed.
      • No need to provision storage in advance.
    • Performance Mode
      • General Purpose - default, suitable for most workloads.
      • Max I/O - higher throughput and IOPS for large-scale applications.
    • Throughput Mode
      • Bursty - default, suitable for most workloads.
      • Provisioned - allows you to specify a specific throughput level.
      • Elastic - scales throughput based on workloads - up to 3GiBs for reads a 1GiBs for writes
    • Storage Classes
      • Standard - for frequently accessed files.
      • Infrequent Access (IA) - for files accessed less frequently, lower cost.
      • Intelligent Tiering - automatically moves files between Standard and IA based on access patterns.
    • Availability and Durability
      • Data is stored across multiple AZs within a region.
      • Designed for 99.999999999% (11 nines) durability.
  • AMI
    • Built for specific regions and can be copied across regions.
    • Amazon Machine Image - template for creating EC2 instances.
    • Includes OS, application server, and applications.
    • Can be customized and shared.
    • Types of AMIs
      • Amazon Provided AMIs
      • AWS Marketplace AMIs
      • Community AMIs
      • Custom AMIs
  • EBS vs EFS
    • EBS is block storage, while EFS is file storage.
    • EBS is attached to a single EC2 instance, while EFS can be mounted on multiple instances.
    • EBS is suitable for high-performance applications, while EFS is ideal for shared file storage.
